QUICK START
-----------

1. Create your first reminder:
   /reminder name:"Morning workout" time:"07:00" day:"Monday" timezone:"America/New_York" ping:true

2. View all reminders:
   /list

3. Check your statistics:
   /stats

SETUP INSTRUCTIONS
------------------

BEFORE YOU START:
1. Ensure the bot is online in your server
2. Check that you have permission to use slash commands
3. Make sure DMs from server members are enabled (for reminder notifications)

GETTING YOUR TIMEZONE:
1. Find your timezone from the list in section 4
2. Remember the exact name (e.g., "America/New_York", not "EST")
3. This ensures reminders trigger at the correct local time

UNDERSTANDING TIME FORMAT:
- Use 24-hour format: HH:MM
- Examples: 09:00 (9 AM), 14:30 (2:30 PM), 23:45 (11:45 PM)
- Invalid: 9:00 AM, 2:30 PM, 24:00, 12:60

FIRST REMINDER EXAMPLE:
/reminder name:"Test reminder" time:"15:30" day:"Today" timezone:"UTC" ping:true
This creates a test reminder for 3:30 PM UTC today with ping enabled.

COMMAND DETAILS
==============

/reminder - Create New Reminder
--------------------------------
Required Parameters:
- name: Description/name for your reminder (max 100 characters)
- time: Time in HH:MM format (24-hour format, e.g., 14:30)
- day: Day of the week (Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday)
- timezone: Your timezone from the dropdown menu
- ping: Whether to ping you when reminder triggers (true/false)

STEP-BY-STEP EXAMPLES:

Example 1 - Work Meeting:
/reminder name:"Team standup meeting" time:"09:30" day:"Monday" timezone:"America/New_York" ping:true
→ Creates a reminder for Monday 9:30 AM Eastern Time with ping enabled

Example 2 - Personal Reminder:
/reminder name:"Take vitamins" time:"08:00" day:"Daily" timezone:"Europe/London" ping:false
→ Creates a daily reminder for 8:00 AM London time without ping

Example 3 - Weekly Task:
/reminder name:"Submit weekly report" time:"17:00" day:"Friday" timezone:"UTC" ping:true
→ Creates a Friday 5:00 PM UTC reminder with ping

DETAILED PARAMETER EXPLANATION:

NAME PARAMETER:
- Max 100 characters
- Be descriptive but concise
- Examples: "Morning workout", "Call mom", "Team meeting", "Take medication"
- Avoid special characters that might cause issues

TIME PARAMETER:
- Format: HH:MM (24-hour clock)
- Hours: 00-23, Minutes: 00-59
- Examples: "06:00" (6 AM), "12:30" (12:30 PM), "18:45" (6:45 PM), "23:59" (11:59 PM)
- Common mistakes to avoid: "6:00", "6:00 AM", "24:00", "12:60"

DAY PARAMETER:
- Select from dropdown: Monday, Tuesday, Wednesday, Thursday, Friday, Saturday, Sunday
- Each reminder is tied to a specific day
- Use repeat functionality for multiple days

TIMEZONE PARAMETER:
- Choose from 20+ available timezones
- Important for accurate timing
- Examples: "America/New_York", "Europe/London", "Asia/Tokyo", "UTC"

PING PARAMETER:
- true: Bot will @mention you when reminder triggers
- false: Bot sends reminder without mention
- Recommended for important reminders
- Consider server etiquette for frequent reminders

COMMON VALIDATION RULES:
- Time must be in 24-hour format (HH:MM)
- Invalid times like "25:99" or "12:60" will be rejected with clear error messages
- Time cannot be in the past for the selected day
- Duplicate reminders with identical settings are not allowed
- Each user can have unlimited reminders

/remove - Remove Specific Reminder
----------------------------------
Select a reminder from the autocomplete list to remove it permanently.

HOW TO USE:
1. Type /remove
2. Start typing the reminder name or time
3. Select from the autocomplete dropdown
4. Confirm removal

AUTOCOMPLETE FORMAT:
Reminders appear as: "Name - Time on Day"
Examples:
- "Morning workout - 07:00 on Monday"
- "Team meeting - 14:30 on Wednesday"
- "Take medication - 20:00 on Daily"

EXAMPLE:
/remove reminder:"Morning workout - 07:00 on Monday"

WHAT HAPPENS:
- Reminder is immediately deactivated
- Won't appear in /list anymore
- Won't trigger notifications
- Statistics remain intact
- Can be recreated with /reminder

SAFETY FEATURES:
- Only your reminders appear in autocomplete
- Confirmation not required (instant removal)
- No way to undo (be careful!)

STEP-BY-STEP PROCESS:

WHAT GETS DELETED:

WHAT REMAINS:

USE CASES:

/edit - Modify Existing Reminder
---------------------------------
Modify specific aspects of existing reminders without recreating them.

PARAMETERS:
- reminder: Select reminder to edit (autocomplete)
- time: New time in HH:MM format (optional)
- day: New day of the week (optional)
- timezone: New timezone (optional)

DETAILED EXAMPLES:

Change Time Only:
/edit reminder:"Team meeting - 14:30 on Wednesday" time:"16:00"
→ Changes meeting from 2:30 PM to 4:00 PM, keeps same day and timezone

Change Day Only:
/edit reminder:"Team meeting - 14:30 on Wednesday" day:"Thursday"
→ Moves meeting to Thursday, keeps same time and timezone

Change Timezone Only:
/edit reminder:"Team meeting - 14:30 on Wednesday" timezone:"Europe/London"
→ Changes timezone, time appears different in local time

Multiple Changes:
/edit reminder:"Team meeting - 14:30 on Wednesday" time:"16:00" day:"Friday" timezone:"UTC"
→ Changes time, day, and timezone all at once

IMPORTANT FEATURES:
- Only modify the fields you want to change
- Unspecified fields remain unchanged
- All validation rules apply (time format, past time check, etc.)
- Autocomplete shows only your reminders
- Changes take effect immediately

VALIDATION RULES:
- Time must be valid HH:MM format
- New time cannot be in past for selected day
- Timezone must be from supported list
- Day must be valid weekday

/repeat - Set Repeat Days
--------------------------
Set reminders to repeat on multiple days while maintaining the same time.

PARAMETERS:
- reminder: Select reminder to modify (autocomplete)
- days: Select day to add/remove from repeat schedule

HOW IT WORKS:
- If the day is NOT in the repeat list, it gets ADDED
- If the day IS already in the repeat list, it gets REMOVED
- You can run this command multiple times to build your repeat schedule
- Original day is preserved unless specifically removed

STEP-BY-STEP EXAMPLES:

Start with single reminder:
/reminder name:"Morning workout" time:"07:00" day:"Monday" timezone:"America/New_York" ping:true

Add Tuesday:
/repeat reminder:"Morning workout - 07:00 on Monday" days:"Tuesday"
→ Now repeats Monday AND Tuesday

Add Wednesday:
/repeat reminder:"Morning workout - 07:00 on Monday" days:"Wednesday"
→ Now repeats Monday, Tuesday, AND Wednesday

Add remaining weekdays:
/repeat reminder:"Morning workout - 07:00 on Monday" days:"Thursday"
/repeat reminder:"Morning workout - 07:00 on Monday" days:"Friday"
→ Now repeats Monday through Friday

Remove Monday:
/repeat reminder:"Morning workout - 07:00 on Monday" days:"Monday"
→ Now repeats Tuesday through Friday only

ADVANCED FEATURES:
- Unlimited repeat days per reminder
- Each repeat maintains same time and timezone
- Original reminder settings preserved
- Can mix and match any days
- Easy to add/remove individual days

COMMON USE CASES:
- Daily habits: Add all 7 days
- Work week: Add Monday-Friday
- Weekends: Add Saturday-Sunday
- Specific schedule: Add only needed days

ERROR HANDLING
--------------

COMPREHENSIVE ERROR GUIDE:

"Invalid time format"
PROBLEM: Time entered doesn't match HH:MM 24-hour format
SOLUTIONS:
- Use HH:MM format (24-hour clock)
- Valid examples: 09:00, 14:30, 23:45
- Invalid examples: 9am, 2:30 PM, 24:00, 12:60, 25:99
- Hours must be 00-23, minutes must be 00-59

COMMON MISTAKES:
- Using 12-hour format with AM/PM
- Single-digit hours (use 09:00, not 9:00)
- Invalid minute values (60+ minutes)
- 24:00 (should be 00:00 for midnight)

"Time is in the past"
PROBLEM: Selected time has already passed for the chosen day
SOLUTIONS:
- Select a future time for the chosen day
- Consider the next occurrence of that day
- Check your timezone setting
- For today, choose a time later than current time

EXAMPLES:
Current time: 2:30 PM Monday
Invalid: "14:00 Monday" (already passed)
Valid: "15:30 Monday" (future time)
Valid: "14:00 Tuesday" (next day)

"Duplicate reminder exists"
PROBLEM: Identical reminder already exists
SOLUTIONS:
- Change reminder name, time, or settings
- Each reminder must be unique in all aspects
- Use /edit to modify existing reminders instead
- Add small details to make it unique

DUPLICATE CRITERIA:
- Same name
- Same time
- Same day
- Same timezone
- Same ping setting
- Same repeat days

"Command not found"
PROBLEM: Slash commands not available
SOLUTIONS:
- Wait a few seconds after bot starts
- Check if bot has proper permissions
- Ensure slash commands are enabled in server
- Try reinviting the bot if needed

PERMISSION CHECKLIST:
- Bot has "Send Messages" permission
- Bot has "Use Slash Commands" permission
- Slash commands enabled in server settings
- Bot is online and responsive

"Reminder not found"
PROBLEM: Trying to edit/remove non-existent reminder
SOLUTIONS:
- Use /list to see your active reminders
- Check reminder ID is correct
- Ensure reminder hasn't been removed
- Reminder must be active to edit

"Invalid timezone"
PROBLEM: Selected timezone not supported
SOLUTIONS:
- Choose from dropdown menu only
- Don't manually type timezone names
- Check timezone list in this guide
- Use standard timezone names

"Autocomplete not working"
PROBLEM: Command suggestions not appearing
SOLUTIONS:
- Start typing to trigger autocomplete
- Wait for suggestions to load
- Check internet connection
- Ensure bot has proper permissions

DATABASE ERRORS:
"Failed to create reminder"
- Try again in a few moments
- Check if bot is online
- Contact support if persistent

"Failed to update reminder"
- Verify reminder still exists
- Check your input values
- Try creating new reminder instead

VALIDATION RULES SUMMARY:
- Time: HH:MM format, 00:00-23:59
- Name: 1-100 characters, no special characters
- Day: Must be valid weekday
- Timezone: Must be from supported list
- Ping: Must be true or false
- No duplicate reminders allowed

TROUBLESHOOTING
---------------

COMPREHENSIVE PROBLEM-SOLVING GUIDE:

BOT NOT RESPONDING:
SYMPTOMS:
- Commands don't work
- No response from bot
- Bot appears offline

CHECKLIST:
1. Verify bot is online
   - Check bot's status in Discord
   - Look for green circle next to bot name
   - Bot should show "Online" status

2. Check bot permissions
   - Go to Server Settings → Roles
   - Verify bot has "Send Messages" permission
   - Ensure "Use Slash Commands" is enabled
   - Check "Read Message History" permission

3. Verify slash commands
   - Type "/" to see command list
   - Commands should appear in dropdown
   - Ensure you're typing in correct channel

4. Check bot connectivity
   - Can other users use commands?
   - Try in different channel
   - Check if bot was restarted

REMINDERS NOT TRIGGERING:
SYMPTOMS:
- No reminder notifications
- Reminders appear to skip
- Time seems wrong

CHECKLIST:
1. Verify bot is running and online
   - Bot must be continuously running
   - Check console for errors
   - Ensure no crashes occurred

2. Check your timezone setting
   - Verify timezone is correct for your location
   - Test with a simple reminder
   - Confirm time matches local time

3. Ensure reminder time hasn't passed
   - Check if time was in the past
   - Verify day selection is correct
   - Consider next occurrence

4. Check reminder status
   - Use /list to verify reminder is active
   - Ensure reminder wasn't accidentally removed
   - Check repeat settings if applicable

5. Verify scheduling system
   - Bot checks reminders every minute
   - System handles timezone conversions
   - Automatic recovery after restarts

DM NOT WORKING:
SYMPTOMS:
- No direct messages from bot
- Reminders only appear in channels
- Error messages about DM failure

SOLUTIONS:
1. Enable DMs from server members
   - Discord Settings → Privacy & Safety
   - Enable "Allow direct messages from server members"
   - Toggle "Keep direct messages from server members"

2. Check privacy settings
   - Server Settings → Privacy Settings
   - Ensure DMs aren't blocked
   - Check "Who can send you a direct message"

3. Verify bot isn't blocked
   - Privacy & Safety → Blocked Users
   - Remove bot from blocked list if present
   - Restart Discord after changes

4. Fallback system
   - Bot automatically sends to channel if DM fails
   - You'll be pinged in the channel
   - All functionality preserved

BUTTONS NOT WORKING:
SYMPTOMS:
- Can't click reminder buttons
- Buttons appear disabled
- "Interaction failed" messages

SOLUTIONS:
1. Verify ownership
   - Only reminder owner can click buttons
   - Check if you created the reminder
   - Use /list to confirm ownership

2. Check button timeout
   - Buttons expire after 24 hours
   - Old reminders show disabled buttons
   - Create new reminder if needed

3. Refresh reminder
   - Edit reminder to reset buttons
   - Or create new reminder
   - Check for system updates

4. Permission issues
   - Ensure bot has "Add Reactions" permission
   - Check "Use External Emojis" permission
   - Verify channel allows embeds

PERFORMANCE ISSUES:
SYMPTOMS:
- Slow command responses
- Delayed reminders
- Bot lagging

OPTIMIZATION:
1. Check bot load
   - High server usage can slow responses
   - Multiple reminders may cause delays
   - Monitor system resources

2. Network issues
   - Check internet connection
   - Verify Discord API status
   - Consider bot hosting quality

3. Database performance
   - Large reminder counts may slow queries
   - Regular cleanup helps performance
   - Monitor database size

ADVANCED TROUBLESHOOTING:

LOG ANALYSIS:

RECOVERY PROCEDURES:
